# 短视频矩阵自动化（抖音/快手/小红书/视频号）

短视频运营者每天重复同一套流程：追热点、写脚本、找素材、剪辑、加字幕、导出多个比例、逐平台发布。一条视频从构思到四平台分发，熟练者也要 4-6 小时。如果同时运营 3 个账号，一天的时间全部搭进去。

这个用例通过 OpenClaw 编排国产大模型 + 剪映 + RPA/API，搭建"脚本生成 → 素材采集 → 视频合成 → 多平台分发"的全链路自动化流水线，将单条视频的制作时间压缩到 10 分钟以内。

> **与小红书用例的区别**：已有的 [cn-xiaohongshu-automation](cn-xiaohongshu-automation.md) 专注**图文笔记**的生成与发布。本用例专注**视频**全链路——从分镜脚本、TTS 配音、素材剪辑到多平台视频分发，技术栈和工作流完全不同。

## 它能做什么

- **热点追踪**：自动抓取抖音、快手、小红书的实时热榜，筛选与账号定位匹配的选题
- **脚本与分镜生成**：多模型协作生成口播文案、分镜描述、字幕文本
- **TTS 配音**：调用 Edge-TTS 或国产语音引擎生成自然语音
- **素材采集与合成**：自动匹配无版权视频素材，通过 FFmpeg 或剪映草稿完成粗剪
- **多版本导出**：从同一草稿导出 9:16（抖音/快手）、3:4（小红书）、16:9（视频号横版）等不同比例
- **四平台定时发布**：通过 API 或 RPA 自动发布到抖音、快手、小红书、视频号
- **数据回收与爆款复制**：监控各平台播放量和完播率，自动将高数据内容裂变为多条变体

## 核心技术方案

### 国产模型分工

整条流水线可以让不同模型各司其职，而非全部依赖单一模型：

| 环节 | 推荐模型 | 选择理由 |
|------|---------|---------|
| 行业资料阅读 | Kimi | 超长上下文（支持 100 页 PDF） |
| 视频逻辑结构设计 | DeepSeek | 深度推理能力强、成本低 |
| 口播文案撰写 | 豆包 | 网感好，擅长口语化表达 |
| 分镜提示词生成 | 通义千问 | 视觉理解与代码生成能力 |
| 数据复盘与归因 | 文心一言 | 知识库丰富，分析能力扎实 |
| 工具调用与编排 | 智谱 GLM | Function calling 支持好（备选） |

OpenClaw 作为编排层，在一条 Skill 链中按阶段调用不同模型的 API。

### 视频合成方案：FFmpeg vs 剪映草稿

**方案 A — FFmpeg 全自动合成（推荐起步）**

通过 OpenClaw 调用 FFmpeg 命令行完成视频处理：

```bash
# 合成示例：视频 + 字幕 + 背景音乐
ffmpeg -i input.mp4 -i bgm.mp3 \
  -vf "subtitles=subs.srt:force_style='FontSize=24'" \
  -c:v libx264 -preset fast -s 1080x1920 \
  -shortest output.mp4
```

优点：全自动、跨平台、免费。缺点：转场和特效能力有限。

**方案 B — 剪映草稿注入（进阶方案）**

通过 Python 脚本直接写入剪映的工程文件（`draft_content.json`），生成可被剪映识别的草稿：

- 工程路径：`AppData/Local/JianyingPro/User Data/Projects/`（Windows）
- 将音频、视频素材自动排列到时间轴
- 在剪映中打开草稿，人工微调转场和封面后导出

优点：可利用剪映的智能字幕、贴纸、卡点音乐。缺点：需要 Windows + 剪映桌面版，且最终导出仍需人工介入。

### 多平台分发方案对比

| 平台 | 官方 API | RPA 方案 | 推荐方式 | 备注 |
|------|---------|---------|---------|------|
| 抖音 | open.douyin.com（仅企业主体） | Airtest 真机/云手机 | 企业用 API，个人用 RPA | API 需营业执照 + 审核 3-5 天 |
| 快手 | open.kuaishou.com（仅企业主体） | Airtest 真机/云手机 | 同上 | 与抖音类似的企业资质要求 |
| 小红书 | 无官方发布 API | Chrome CDP + Python | RPA | 仅支持图文 API，视频需 RPA |
| 视频号 | 微信视频号助手（受限） | 网页端自动化 | 视频号助手 + RPA | 敏感词审核最严格 |

> **关键事实**：抖音和快手的视频发布 API 仅对**企业主体**开放，个人开发者无法申请。个人用户只能通过 RPA（真机模拟操作）或手动发布。

## 如何设置

### 前置条件

- OpenClaw 已安装并配置
- Python 3.10+、FFmpeg、Node.js
- 各平台账号已注册
- （可选）云手机服务（用于 RPA 多账号隔离）

### 第一步：配置模型 API

在 OpenClaw 中配置需要调用的国产模型（至少配置一个用于文案生成）：

```yaml
# 示例：在 OpenClaw 配置中添加模型
models:
  deepseek:
    api_key: "sk-xxx"
    base_url: "https://api.deepseek.com/v1"
  doubao:
    api_key: "xxx"
    base_url: "https://ark.cn-beijing.volces.com/api/v3"
```

### 第二步：搭建视频生成 Skill

创建一个 OpenClaw Skill，将脚本生成、TTS、素材匹配、FFmpeg 合成串联起来：

```text
# 对 OpenClaw 下达指令示例
帮我生成一条关于"居家健身"的短视频：
1. 先写一段 30 秒口播脚本，要有 hook 开头
2. 用 Edge-TTS 生成配音
3. 从素材库匹配 3-5 段健身相关视频片段
4. 用 FFmpeg 合成：加字幕、背景音乐、导出 9:16 竖版
```

### 第三步：配置多平台发布

**企业用户（有营业执照）**：通过 Composio MCP 或官方 SDK 接入抖音/快手 API，实现一键发布。

**个人用户**：使用 Airtest 或 Chrome CDP 编写 RPA 脚本，模拟人工操作发布。建议在 OpenClaw 中封装为独立的发布 Skill。

### 第四步：设置定时调度

利用 cron 或 OpenClaw 的定时任务功能，按最优时间窗口自动触发：

- **7:00-9:00**：早通勤（知识/资讯类）
- **12:00-14:00**：午休（娱乐/搞笑类）
- **20:00-22:00**：晚间黄金档（全品类）

## 实用建议

- **先跑通单平台再扩展**：建议从抖音开始验证全流程，再复制到其他平台
- **半自动优于全自动**：纯 AI 拼接的视频容易被平台判定为低质内容。保留人工审核封面、转场等关键环节，完播率和推荐量明显更高
- **内容去重是必修课**：同一素材发多平台必须做差异化处理——调整亮度（±5%）、播放速度（1.0x vs 1.05x）、镜像翻转、替换 BGM、修改元数据
- **一机一号一 IP**：RPA 方案务必做设备指纹隔离，多账号共用设备是封号第一大原因
- **文案差异化**：不同平台使用不同模型生成标题和描述，避免文本查重
- **控制发布频率**：每账号每天不超过 5 条，间隔 15-30 分钟随机延迟

> **⚠️ 平台风控提醒**：抖音、快手、小红书、视频号均对自动化操作有严格的风控策略。违规风险包括但不限于：
>
> - **账号限流或封禁**：批量发布、设备指纹异常、内容重复度高均可能触发
> - **内容审核驳回**：AI 生成内容如果缺乏原创性，可能被判定为"搬运"而限制推荐
> - **API 权限撤销**：违反平台开发者协议可能导致 API 资质被收回
> - **法律风险**：使用未授权素材、虚假宣传等可能面临法律责任
>
> **强烈建议**：先用测试账号充分验证，始终保留人工审核环节，以辅助创作为目标而非批量灌水。遵守各平台的社区规范和开发者协议。

## 适用场景

- 好物推荐、影视解说、知识科普等**格式化内容**的批量生产
- MCN / 自媒体工作室的**多账号矩阵运营**
- 品牌方的**多平台内容分发**（需企业资质）
- 个人创作者的**效率提升**（减少重复劳动，聚焦创意）

## 不适用场景

- 需要高度原创创意或强人格化的内容（如 vlog、真人出镜）
- 对画面质量有极高要求的专业影视制作
- 直播相关场景（本用例仅覆盖短视频）
