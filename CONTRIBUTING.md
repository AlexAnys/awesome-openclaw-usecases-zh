# 贡献指南

感谢你对本项目的关注！我们欢迎各种形式的贡献。

## 添加用例

1. 在 `/usecases` 文件夹中创建一个以你的用例命名的 Markdown 文件
2. 详细描述你的用例。建议涵盖以下要点：
   - **痛点** — 这个用例解决什么问题？
   - **功能说明** — 简要说明这个用例做什么
   - **提示词** — 你给 OpenClaw 的实际提示词/配置
   - **所需技能** — 需要哪些 OpenClaw 技能
   - **相关链接** — 相关文档、API、工具
3. 在主 [README.md](README.md) 的对应分类表格中添加一行
4. 提交 Pull Request

## 贡献类型

### 翻译改进
如果你发现现有翻译有不准确或不自然的地方，欢迎提交修改。

### 中文独创用例
我们特别鼓励提交中文社区独创的用例！如果你发现了 OpenClaw 在中文场景下的独特用法（比如接入微信、钉钉、飞书等中文平台），请直接提交。

### 改进现有用例
如果你对某个用例有更好的实践经验或补充说明，欢迎改进。

## 规范

- 每个 Markdown 文件一个用例
- 描述要简洁，但足够让别人复现
- 如果你的用例不属于现有分类，可以在 PR 中建议新分类
- 允许重复主题，前提是方法有显著不同
- 不要用 AI 凭空生成用例。只提交你实际测试和验证过的用例
- **不接受加密货币相关用例**

## 翻译规范

如果你在翻译或改进翻译，请遵循以下规范：

- 保持原有 Markdown 结构和格式
- 标题和正文全部使用中文
- 代码块中的注释翻译为中文，代码/命令保留英文
- prompt 示例保留英文原文，上方添加中文说明
- 技术术语首次出现时添加括号注释（如 webhook（网络钩子））
- 每个翻译文件底部保留原文链接
